## 四、最佳实践（沉淀）

### 1. 自动化交付的标准动作
```
推送代码前：
  ✅ python -m py_compile 语法检查
  ✅ 确认 GitHub 连通性（curl -s https://github.com）
  
推送代码后：
  ✅ 手动触发 Run Workflow
  ✅ 等待 workflow 完成
  ✅ 检查每一步的结论（success/failure）
  ✅ 有失败立即定位根因并修复
```

### 2. 多步沉淀机制
发现任何一个技术根因后，必须三步同时做：
```
1. 写日志（今日日志 + 项目日志）
2. 存记忆（mem0）
3. 推存档（本地 devlog → GitHub 开发日志）
```

### 3. 自动化架构的边界
GitHub Actions 已有完整的流水线 → AI 只需触发，不要手动手动

### 4. 数据驱动归因
不要靠"感觉好像经常失败"，而是从数据库拉统计数据说话
