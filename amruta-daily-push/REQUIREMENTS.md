## 一、核心需求（主人确认版）

### 1. 标题
- 中文标题优先从 IMA 知识库 pairs 中**关键词匹配**提取
- 找不到 → **阿里云翻译**兜底
- **绝对不要**出现英文标题当中文用的情况

### 2. 正文英文
- **必须和 amruta.today 原文完全一致，不多不少**
- 如果不一致 → 必须修改代码直到一致

### 3. 正文中文
- 优先用 IMA 知识库 BGE 对齐获取官方翻译
- IMA 没有对应内容 → 阿里云翻译兜底（代码实现细节，不做需求显式列出）

### 4. 底部链接
- 优先用 IMA MD 的 `source` 字段 → **sahaja.live**
- 没有 sahaja.live → 降级为 **amruta.today**

### 5. GitHub 存档
- 正常保存，无需特殊处理去重前后问题

### 6. 语义去重
- 短句保护：**< 15 词的英文句不参与语义去重**

### 7. 邮件
- 每天北京时间 04:00 自动发送
- 发送人：Shri Mataji 每日讲话
- 正文格式：中英对照，英文灰色小字 + 中文正常

## 二、检测流程（6.15→6.12）

按日期倒序检测，每天检查以下 **4 项通过标准**：

| # | 检查项 | 通过标准 |
|---|--------|---------|
| A | **中文标题** | 标题是中文（非英文），翻译合理 |
| B | **英文正文一致性** | 存档和邮件中的英文与 amruta.today 原文一字不差 |
| C | **底部链接** | 为 sahaja.live 链接（或降级为 amruta.today） |
| D | **中英对齐正确** | 中文翻译与对应英文句子匹配，无明显错配 |

**检测顺序**：6.15 → 6.14 → 6.13 → 6.12  
**规则**：不达标 → 修改代码 → 重新运行 → 再检测 → 直到达标 → 进入下一天
